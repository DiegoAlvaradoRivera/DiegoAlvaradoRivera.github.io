Descripción de las variables

El siguiente dataset está compuesto de 414 observaciones y 7 atributos.

  • transaction_date: fecha de la transacción (por ejemplo, 2013.250=2013 March, 2013.500=2013 June, etc.)
  • house_age: antigüedad de la casa (años)
  • distance_to_nearest_MRT_station: distancia a la estación MRT más cercana (metros)
  • number_of_convenience_stores: número de tiendas de conveniencia cercanas (unidades)
  • latitude: coordenada geográfica correspondiente a la latitud (grados)
  • longitude: coordenada geográfica correspondiente a la longitud (grados)
  • price_of_unit_area: precio de la casa por unidad de area (10000 dólares taiwaneses/Ping, 1 Ping: 3.3 metros cuadrados)

El objetivo es predecir el precio por unidad de área e identificar cómo las diferentes variables influyen en el precio.

Separación del conjunto de datos en train y test

Se procede a separar el dataset en dos conjuntos - train: este conjunto se utilizará para el entrenamiento los modelos y la optimización de hiperparámetros - test: este conjunto se utilizará para evaluar el rendimiento de los modelos terminados

Para asegurar que los resultados sean significativos, se aplica muestreo estratificado para separar el dataset. Los estratos están determinados por la variable “price_of_unit_area”, la cual se discretiza en los intervalos (-Inf, 30), (30, 50) y (50, +Inf)

El número de observaciones que corresponde a cada dataset es el siguiente: - train: 289 obvs - test: 124 obvs

# Crear el conjunto de datos de entrenamiento

Modelamiento

A continuación se procede a modelar el precio por unidad de área. Se evaluará tres modelos:

  • GAM sin localización geográfica (GAM 1): house age, number of stores, MRT station distance
  • GAM con localización geográfica (GAM 2): house age, latitude, longitude
  • Random Forests (RF): todas las variables

Se considerará modelos GAMs ya que son flexibles, pero interpretables. Adicionalmente, se entrenará un modelo random forest para comparar el rendimiento de los modelos.

Ya que las GAM son modelos lineales, estas son sensibles a la multicolinealidad, la cual puede degradar severamente la estimación de los parámetros del modelo. Como se vio anteriormente, las variables “number_of_conv_stores” y “dist_mre_station” están fuertemente relacionadas con la localización geográfica. Así, estas variables no pueden ser utilizadas de forma conjunta en un mismo modelo.

Entrenamiento del modelo GAM 1

Detalles del modelo:

  • Para capturar el efecto de la localización geofráfica, se utilizará un smooth de dos dimensiones (lat, long)
  • los hiperparámetros se seleccion automaticamente (minimizando el leave-one-out CV error

Validación de los supuestos del modelo:

  • los errores se distribuyen aproximadamente de forma normal, aunque existen algunos errores positivos bastante grandes.
  • El modelo converge y no se observan problemas de concurvidad

Resultados del entrenamiento:

  • todas las variables incluidas tienen efectos significativos en el precio

fit_measure <- function(model){
  aic <- AIC(model)                    # AIC
  gcv <- summary(model)$sp.criterion   # GCV
  rsq <- summary(model)$r.sq           # R squared
  
  cat("AIC:", aic, " GCV:", gcv, " R.SQ:", rsq, "\n")
}
