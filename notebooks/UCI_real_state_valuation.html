Descripcin de las variables

El siguiente dataset est compuesto de 414 observaciones y 7 atributos.

  • transaction_date: fecha de la transaccin (por ejemplo, 2013.250=2013 March, 2013.500=2013 June, etc.)
  • house_age: antigedad de la casa (aos)
  • distance_to_nearest_MRT_station: distancia a la estacin MRT ms cercana (metros)
  • number_of_convenience_stores: nmero de tiendas de conveniencia cercanas (unidades)
  • latitude: coordenada geogrfica correspondiente a la latitud (grados)
  • longitude: coordenada geogrfica correspondiente a la longitud (grados)
  • price_of_unit_area: precio de la casa por unidad de area (10000 dlares taiwaneses/Ping, 1 Ping: 3.3 metros cuadrados)

El objetivo es predecir el precio por unidad de rea e identificar cmo las diferentes variables influyen en el precio.

Separacin del conjunto de datos en train y test

Se procede a separar el dataset en dos conjuntos - train: este conjunto se utilizar para el entrenamiento los modelos y la optimizacin de hiperparmetros - test: este conjunto se utilizar para evaluar el rendimiento de los modelos terminados

Para asegurar que los resultados sean significativos, se aplica muestreo estratificado para separar el dataset. Los estratos estn determinados por la variable price_of_unit_area, la cual se discretiza en los intervalos (-Inf, 30), (30, 50) y (50, +Inf)

El nmero de observaciones que corresponde a cada dataset es el siguiente: - train: 289 obvs - test: 124 obvs

Modelamiento

A continuacin se procede a modelar el precio por unidad de rea. Se evaluar tres modelos:

  • GAM sin localizacin geogrfica (GAM 1): house age, number of stores, MRT station distance
  • GAM con localizacin geogrfica (GAM 2): house age, latitude, longitude
  • Random Forests (RF): todas las variables

Se considerar modelos GAMs ya que son flexibles, pero interpretables. Adicionalmente, se entrenar un modelo random forest para comparar el rendimiento de los modelos.

Ya que las GAM son modelos lineales, estas son sensibles a la multicolinealidad, la cual puede degradar severamente la estimacin de los parmetros del modelo. Como se vio anteriormente, las variables number_of_conv_stores y dist_mre_station estn fuertemente relacionadas con la localizacin geogrfica. As, estas variables no pueden ser utilizadas de forma conjunta en un mismo modelo.

Entrenamiento del modelo GAM 1

Detalles del modelo:

  • Para capturar el efecto de la localizacin geofrfica, se utilizar un smooth de dos dimensiones (lat, long)
  • los hiperparmetros se seleccion automaticamente (minimizando el leave-one-out CV error

Validacin de los supuestos del modelo:

  • los errores se distribuyen aproximadamente de forma normal, aunque existen algunos errores positivos bastante grandes.
  • El modelo converge y no se observan problemas de concurvidad

Resultados del entrenamiento:

  • el modelo es altamente significativo (varianza explicada = 60%)
  • todas las variables incluidas tienen efectos significativos en el precio
  • leave-one-out CV error: 60.603

  aic <- AIC(model)                    # AIC
  gcv <- summary(model)$sp.criterion   # GCV
  rsq <- summary(model)$r.sq           # R squared
  
  cat("AIC:", aic, " GCV:", gcv, " R.SQ:", rsq, "\n")
}
